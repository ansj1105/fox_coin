# 친구 초대 & 래퍼럴 구조 (3중 연동)

> **친구 초대 = 채굴 + 수익 + 성장**

메인페이지 채굴·초대 기능 설계 및 구현 기준 문서입니다.

- **채굴 기본 조건, 레벨별 제한, EXP 시스템** → [MINING_AND_LEVEL_SPEC.md](./MINING_AND_LEVEL_SPEC.md) 참고

---

## 1. 친구 초대 → 채굴 속도 보너스

- **일일 최대 채굴량은 유지** (캡 변경 없음)
- **영상 1회당 채굴량**이 초대 인원 구간에 따라 증가

| 초대 인원 | 채굴량 증가율 |
|----------|---------------|
| 1명 | +3% |
| 5명 | +6% |
| 10명 | +10% |
| 20명 | +14% |
| 50명 | +18% |
| 100명 | +22% |

- 적용: 영상 시청 1회당 지급되는 채굴량에 위 비율을 곱하여 보너스 적용 (일일 최대 채굴량 상한은 기존과 동일)

---

## 2. 래퍼럴 수익 (하부 채굴 연동)

- **하부 유저가 실제로 채굴한 KORI** 기준으로 상위 유저에게 수익 지급
- 하부 1명당 채굴량이 아닌, **총 하부 인원 수 구간**에 따라 수익률 적용

| 하부 유저 수 | 수익률 |
|-------------|--------|
| 1~4명 | 3% |
| 5명 | 5% |
| 10명 | 7% |
| 20명 | 9% |
| 50명 | 11% |
| 100명 | 13% |

- 예: 하부 10명이 오늘 채굴한 KORI 합계가 1000 KORI이고, 해당 구간 수익률이 7%이면 상위 유저 수익 = 1000 × 7% = 70 KORI

---

## 3. 래퍼럴 안전장치

| 항목 | 규칙 |
|------|------|
| **하루 유저** | 이메일 인증 필수 |
| **채굴 인정** | 영상 시청 + 채굴 기록 필수 (시청만으로는 인정 안 함) |
| **중복 초대 무효** | 동일 IP / 동일 기기로 여러 계정 초대 시 해당 초대는 무효 처리 |

- 구현 시: IP·기기(디바이스 ID) 중복 체크, 이메일 인증 완료 여부, 영상 시청·채굴 로그 연동 검증 필요

---

## 4. 친구 초대 경험치(EXP) 보너스

- **초대 1명당: 1 EXP** 지급
- 초대한 친구가 가입·인증 완료 등 조건 충족 시 1회만 지급 (안전장치 통과 시)

---

## 5. 메인페이지 구현 포인트

- **채굴 영역**: 현재 채굴량, 일일 한도, **초대 인원 수별 채굴 보너스 %** 표시
- **초대 영역**: 초대 링크/코드, 초대 인원 수, **채굴 보너스 구간 테이블**(1명~100명)
- **래퍼럴 수익 영역**: 하부 채굴 KORI 합계, **수익률 구간 테이블**, 예상/누적 래퍼럴 수익
- **EXP**: 초대 1명당 1 EXP 안내 및 현재 EXP 표시

---

## 6. 요약 표

| 구분 | 내용 |
|------|------|
| 채굴 보너스 | 초대 1/5/10/20/50/100명 → +3%/~+22% (영상 1회당, 일일 상한 유지) |
| 래퍼럴 수익 | 하부 채굴 KORI 기준 3%~13% (하부 1~4명: 3%, 100명: 13%) |
| 안전장치 | 이메일 인증 필수, 영상 시청+채굴 기록 필수, 동일 IP/기기 중복 초대 무효 |
| EXP | 초대 1명당 1 EXP |
